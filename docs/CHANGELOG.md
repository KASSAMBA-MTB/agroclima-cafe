# Política de Versionamento

O projeto utilizará Versionamento Semântico (Semantic Versioning).

Formato:

MAJOR.MINOR.PATCH

Exemplo:

1.0.0

Onde:

* MAJOR → alterações incompatíveis.
* MINOR → novas funcionalidades.
* PATCH → correções e pequenos ajustes.
